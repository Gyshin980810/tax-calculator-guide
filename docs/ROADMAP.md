## Phase 2: 공통 모듈/컴포넌트 개발 (진행 중)

**예상 소요 시간**: 2-3시간

### 수행 작업
- [ ] **Notion API 공통 함수 완성**
  - `getAllGuides()` - 모든 가이드 조회 ✅ (구현 완료)
  - `getGuideById(id)` - ID로 특정 가이드 조회 ✅ (구현 완료)
  - `getGuidesByCategory(category)` - 카테고리별 조회 ✅ (구현 완료)
  - `searchGuides(query)` - 키워드 검색 ✅ (구현 완료)

- [ ] **공통 컴포넌트 개발**
  - Header 컴포넌트 (네비게이션, 로고)
  - Footer 컴포넌트 (저작권, 링크)
  - GuideCard 컴포넌트 (가이드 목록용 카드)
  - CategoryCard 컴포넌트 (홈 화면 카테고리)
  - SearchBox 컴포넌트 (검색 입력)
  - CategoryFilter 컴포넌트 (필터링 UI)

- [ ] **공통 타입 정의** ✅ (완료)
  - `TaxGuide` 타입
  - `TaxCategory` 타입
  - `GuideStatus` 타입
  - 상수 정의 (TAX_CATEGORIES)

- [ ] **레이아웃 구성**
  - Root Layout (Header, Footer, ThemeProvider 포함)
  - 페이지 레이아웃 통일

### 이유
공통으로 사용되는 코드를 먼저 만들어야 각 페이지에서 중복을 방지하고, 일관된 디자인을 유지할 수 있습니다.

### 예상 소요 시간 분배
- Notion 함수: 30분 (이미 구현됨)
- 컴포넌트: 1시간 30분
- 타입 정의: 20분 (이미 구현됨)
- 레이아웃: 40분

### 완료 기준
- [ ] 모든 공통 컴포넌트가 구현되고 테스트됨
- [ ] Storybook 또는 컴포넌트 카탈로그에서 확인 가능
- [ ] `src/lib/notion.ts`의 모든 함수가 에러 처리 포함
- [ ] 타입스크립트 타입 검사 통과 (`npm run type-check`)
- [ ] Header/Footer가 모든 페이지에 일관되게 표시됨

## Phase 3: 핵심 기능 개발

**예상 소요 시간**: 2-3시간

### 수행 작업
- [ ] **홈 페이지 (`/`) 구현**
  - 4가지 세금 유형별 카테고리 카드 표시
  - 최신 업데이트 가이드 섹션
  - 검색 바
  - Hero 섹션 (프로젝트 소개)

- [ ] **카테고리 목록 페이지 (`/category/[name]`) 구현**
  - 선택한 세금 유형의 모든 가이드 목록
  - 각 가이드의 제목, 요약, 수정 날짜 표시
  - 가이드 클릭 시 상세 페이지로 이동

- [ ] **가이드 상세 페이지 (`/guides/[id]`) 구현**
  - 가이드 제목, 카테고리, 수정 날짜 표시
  - 가이드 내용 (Content 필드)
  - 계산 사례 (Example 필드)
  - 체크리스트 (Checklist 필드)
  - 목차 사이드바 (Optional)

- [ ] **Notion 콘텐츠 렌더링**
  - Rich Text 필드 마크다운/HTML 렌더링
  - 이미지 최적화 (Next.js Image 컴포넌트)
  - 코드 블록 하이라이팅

### 이유
홈, 목록, 상세 페이지는 포탈의 가장 기본이 되는 기능이며, 이 3개 페이지가 완성되면 사용자가 기본적인 사용 경험을 할 수 있습니다.

### 예상 소요 시간 분배
- 홈 페이지: 50분
- 카테고리 목록 페이지: 45분
- 상세 페이지: 60분
- 콘텐츠 렌더링: 25분

### 완료 기준
- [ ] 모든 페이지가 정상적으로 렌더링됨
- [ ] Notion API에서 데이터를 정상적으로 불러옴
- [ ] 페이지 간 네비게이션이 정상 작동
- [ ] 모든 필드가 올바르게 표시됨
- [ ] 개발 서버에서 에러 없음 (`npm run dev`)
- [ ] 모바일 화면에서도 기본 레이아웃이 정상 동작

## ⚠️ 주의사항

### 의존성
- **Phase 2** → **Phase 3**: Phase 2의 공통 컴포넌트가 없으면 Phase 3 진행 불가
- **Phase 3** → **Phase 4**: 핵심 기능이 완성되어야 추가 기능 구현 가능
- **Phase 4** → **Phase 5**: 모든 기능이 완성된 후 최적화 진행

### 위험 요소
1. **Notion API 레이트 제한**: 요청이 많으면 API 호출 제한 가능
   - 해결: 캐싱 및 ISR(Incremental Static Regeneration) 활용

2. **풍부한 콘텐츠 렌더링**: Notion Rich Text는 복잡할 수 있음
   - 해결: `notion-to-markdown` 또는 유사 라이브러리 검토

3. **성능**: 초기 로딩 시간이 길 수 있음
   - 해결: SSG, 이미지 최적화, 코드 스플리팅

4. **모바일 반응형**: 초기에 고려하지 않으면 나중에 수정 어려움
   - 해결: Phase 3부터 반응형 먼저 고려
